Problem

Design a URL shortening service like TinyURL/bit.ly that converts long URLs to short ones and redirects.

Architecture

Client → Load Balancer → API Servers → Cache (Redis) ↓ Database (NoSQL) ↓ Analytics Service → Dashboard Key Components: • Base62 encoding for short URL generation • Consistent hashing for DB sharding • Redis cache for hot URLs (80/20 rule) • TTL-based expiration • Analytics pipeline for click tracking

Key Decisions

EncodingBase62 (a-z, A-Z, 0-9) → 6 chars = 56B URLs
DatabaseNoSQL (DynamoDB/Cassandra) for write-heavy workload
CachingRedis with LRU eviction, 80% hit rate
Scale100M URLs/day, 10:1 read-write ratio

LRU Cache
LLD
GoogleAmazonMicrosoft
→

Problem

Design a Least Recently Used cache with O(1) get and put operations.

Design

HashMap + Doubly Linked List HashMap: key → Node (O(1) lookup) DLL: maintains access order (O(1) insert/remove) class LRUCache: def __init__(self, capacity): self.cache = {} self.capacity = capacity self.head = Node(0,0) self.tail = Node(0,0) self.head.next = self.tail self.tail.prev = self.head def get(self, key): # O(1) if key in self.cache: node = self.cache[key] self._remove(node) self._add(node) return node.val return -1 def put(self, key, val): # O(1) if key in self.cache: self._remove(self.cache[key]) node = Node(key, val) self._add(node) self.cache[key] = node if len(self.cache) > self.capacity: lru = self.head.next self._remove(lru) del self.cache[lru.key]
